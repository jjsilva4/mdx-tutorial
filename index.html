Set up mdx-deck

System requirements:

  • Install Yarn on your machine
  • File editor program such VS Code or Atom
  • Basic understanding of Markdown

How to run MDX deck

  • Open the command line or terminal on your machine and run this command: npm i -D mdx-deck
  • Open Finder/File Explorer and create a folder for your presentation.
  • Take note of the folder's path üìù
  • Inside your new folder, create a .MDX file with a relevant name. Ex) deck.mdx
  • For testing, try pasting code from blog post into your deck.mdx file and save
  • In the command line, use the cd command to change your directory to your presentation‚Äôs folder.

ex) cd ~/documents/how-install-mdx-presentation

  • Then run yarn init to create the package.json file.
  • Open the package.json file in VS Code (or other editor of choice) and paste this code.

"scripts": { "start": "mdx-deck deck.mdx" }

Note: Don't forget the comma and don't forget to update presentation name. See blog post if you need help.

  • Run yarn start in the command line.

With this command, your browser will automatically open a new tab with the presentation slides. Mine opened at http://localhost:8080/#0 but your port number may differ.

Edit your .mdx file to build out the slides, and use the browser window to present.

Themes

Customize your presentation's theme by editing the .mdx file and changing the first line of code.

export { notes as theme } from 'mdx-deck/themes'

Export to PDF

You‚Äôll need to add another line of code to your package.json file to access the yarn pdf command. Here is an example:

"scripts": { "start": "mdx-deck deck.mdx", "pdf": "mdx-deck pdf deck.mdx" }

If you get an error after running yarn pdf, first check that you changed the directory in the command line using the cd operation.

Host presentation online

Use the yarn build command to export a .html and .js file of your presentation, after adding the build line of code. I added it to end of my script section:

"scripts": { "start": "mdx-deck deck.mdx", "pdf": "mdx-deck pdf deck.mdx", "build": "mdx-deck build deck.mdx" }

Once you save your package.json file, you may now access the yarn build command in the command line. Again, make sure you use the cd command to point your terminal to correct folder before you run this yarn build command. When the build finishes, you will find the .html and .js files in the dist folder of your mdx-deck project folder.
